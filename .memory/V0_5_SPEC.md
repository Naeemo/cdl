# CDL v0.5 语法规范
**设计日期**: 2026-03-13  
**目标**: 像 Markdown 一样简单，2 分钟画出第一个图表  
**原则**: 语法元素 ≤ 10 个，无需学习查询语言

---

## 🎯 核心设计

### 三大规则

1. **用 `#` 定义图表标题**
   ```cdl
   # 月度销售趋势
   ```
   自动创建名为"月度销售趋势"的 Chart

2. **用 Markdown 表格定义数据**
   ```cdl
   | 月份 | 销售额 |
   | 1月  | 100    |
   | 2月  | 150    |
   | 3月  | 200    |
   ```
   自动生成 DataDefinition，列名即字段名

3. **用 `##` 指定图表类型（可选）**
   ```cdl
   ## 折线图
   ```
   或自动推断（见下文）

---

## 📝 完整示例

### 折线图（自动推断）
```cdl
# 月度销售趋势

| 月份 | 销售额 |
| 1月  | 100    |
| 2月  | 150    |
| 3月  | 200    |
```
**推断**：标题含"趋势" → type: line

## 🔀 图表类型映射

| 关键词（不区分大小写） | Chart.type | 适用场景 |
|----------------------|------------|----------|
| 折线图、趋势、变化 | line | 时间序列、趋势 |
| 柱状图、柱形图、条形图、对比 | bar | 类别对比、排名 |
| 饼图、占比、构成 | pie | 占比分析 |
| 散点图、分布、关系 | scatter | 相关性分析 |
| 面积图、累积 | area | 累积趋势 |
| 雷达图、能力模型 | radar | 多维度评估 |
| 热力图、矩阵 | heatmap | 二维矩阵 |
| 组合图、混合 | combo | 多序列混合 |

**推断逻辑**：
1. 如果存在 `## 图表类型` 指令，优先使用
2. 否则从 `# 标题` 中匹配关键词
3. 默认 `line`（折线图）

## 🎨 样式定制（可选）

在图表定义后添加 `@` 指令：

```cdl
# 月度销售趋势

| 月份 | 销售额 |
| 1月  | 100    |
| 2月  | 150    |
| 3月  | 200    |

## 折线图
@color "#4fc3f7"
@title "2024年销售走势"
@style "平滑曲线"
```

**支持的 @ 指令**：
- `@color "<hex>"` - 主色调
- `@title "<text>"` - 覆盖标题
- `@style "<desc>"` - 样式描述（由渲染器解释）
- `@animation "<effect>"` - 动画效果

## 📊 多图表支持

一个文件可定义多个图表，用 `---` 分隔：

```cdl
# 月度销售趋势

| 月份 | 销售额 |
| 1月  | 100    |
| 2月  | 150    |
| 3月  | 200    |

## 折线图

---

# 区域销售对比

| 区域 | 销售额 |
| 华北 | 120    |
| 华南 | 200    |
| 华东 | 180    |

## 柱状图
@color "#667eea"
```

## 🔄 向后兼容 v0.1

v0.5 是默认语法，但 v0.1 仍可通过以下方式使用：

**方式 1：文件扩展名**
- `.cdl` - v0.5（默认）
- `.cdl.v1` - 强制 v0.1 语法

**方式 2：CLI 标志**
```bash
cdl compile chart.cdl --v0.1
```

**方式 3：首行声明**（可选）
```cdl
@cdl v0.1
// 后续使用 v0.1 语法
```

## 🆚 v0.1 vs v0.5 对比

| 特性 | v0.1 (高级模式) | v0.5 (极简模式) |
|------|----------------|----------------|
| 学习成本 | 高（需学多种语法） | 极低（只需表格） |
| 数据源 | SQL/DAX/Data | 仅 Markdown 表格 |
| 图表定义 | 显式 Data + Chart 块 | 标题 + 表格 + 类型推断 |
| 灵活性 | 高（可复杂查询） | 中（适合静态数据） |
| 适用场景 | 生产环境、复杂查询 | 快速原型、文档、博客 |
| 用户画像 | 开发者、分析师 | 所有人（包括非技术）|

## 🎓 迁移指南（v0.1 → v0.5）

**v0.1 示例**：
```cdl
@lang(sql)
@source('sales_db')
SalesData {
    SELECT month, amount FROM sales WHERE year = 2024
}

Chart 月度销售 {
    use SalesData
    type line
    x month
    y amount
}
```

**等价 v0.5**：
```cdl
# 月度销售

| 月份 | 销售额 |
| 1月  | 100    |
| 2月  | 150    |
| 3月  | 200    |

## 折线图
```
（注意：v0.5 适合静态数据，SQL 查询需要在运行时注入数据）

## ⚠️ 限制与约束

v0.5  intentionally 限制：
- ❌ 不支持 SQL/DAX 内联（数据必须是表格字面量）
- ❌ 不支持 `@source`, `@timeout`, `@cache`（配置外置）
- ❌ 不支持 `group`, `stack`, `where`（需预处理数据）
- ✅ 16+ 图表类型全部支持
- ✅ 样式定制（@color, @title, @style）

**哲学**：v0.5 不是"功能削减"，而是**场景聚焦**——为"快速分享图表"而生。复杂需求用 v0.1。
